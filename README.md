# 📧 Email Knowledge Extraction & Graph Memory

This project ingests raw email data, extracts structured “claims” and entities using an LLM, normalises and de-duplicates the results, and persists everything in a Neo4j knowledge graph. A simple QA layer lets you query the accumulated memory by keywords.

---

## 🔍 What it does

1. **Load** a CSV of emails (`sample_5.csv`, `emails.csv`, etc.).
2. **Parse** the raw MIME text; clean quoted text and forwarded sections.
3. **Call an LLM** (via an Ollama local server) to extract:

   * entities (`Person`, `Organization`, `Project`, `Topic`)
   * claims (`Decision`, `RoleAssignment`, `MeetingPlan`, etc.)
   * evidence spans with character offsets
4. **Normalise / verify** claims and entity data; deduplicate.
5. **Hash** each email body into an artifact ID.
6. **Persist** artifacts, entities, claims and evidence into Neo4j.
7. **Resume-safe processing** – progress tracked in `progress.json`.
8. **Query** the graph with simple keyword searches or run the QA pipeline.

---

## 📂 Dataset

This project is designed to work with the **Enron Email Dataset**:

🔗 [https://www.kaggle.com/datasets/wcukierski/enron-email-dataset](https://www.kaggle.com/datasets/wcukierski/enron-email-dataset)

The dataset contains approximately **500,000 real emails** generated by Enron employees.

### Expected format

Your CSV should include a column containing full RFC-822 raw email text, typically:

* `message`
* or `content`

The system parses:

* Subject
* Sender
* Timestamp
* Body

from the raw MIME content.

## ⚙️ Requirements

* Python 3.11+
* Neo4j (bolt://localhost:7687 by default)
* [Ollama](https://ollama.com) running locally with a model (default: `mistral:latest`)
* Required Python packages:

```bash
pip install pandas neo4j requests pydantic pytz
```

You can generate a `requirements.txt` with:

```bash
pip freeze > requirements.txt
```

## 🚀 Getting started

### 1️⃣ Start Neo4j

Ensure Neo4j is running.

Default credentials used in code:

```
username: neo4j
password: asdfghjkl
```

(Adjust in `main.py` if needed.)

---

### 2️⃣ Launch Ollama

Start Ollama and pull a model:

```bash
ollama pull mistral
```

Then run the server (if not already running):

```bash
ollama serve
```

---

### 3️⃣ Prepare the dataset

Download the Enron dataset from Kaggle.

Convert it to a CSV with a `message` column containing full raw email text if needed.

Update the path in `main.py`:

```python
CSV_PATH = "emails.csv"
```

---

### 4️⃣ Run ingestion

```bash
python main.py
```

Progress is stored in `progress.json` so you can safely stop and resume.

## 🧱 Core modules

| File               | Purpose                                                 |
| ------------------ | ------------------------------------------------------- |
| `extraction.py`    | Load emails, clean body, call LLM and validate output   |
| `prompts.py`       | LLM prompt template enforcing strict extraction rules   |
| `normalizer.py`    | String normalization, deduplication, evidence fixing    |
| `models.py`        | Pydantic models for entities/claims/evidence            |
| `graph_builder.py` | Neo4j wrapper for inserting artifacts, entities, claims |
| `deduper.py`       | Utility to merge duplicate claims                       |
| `qa_pipeline.py`   | Keyword-based search & simple QA over the graph         |
| `query.py`         | Example usage of `MemoryQA` class                       |
| `main.py`          | Resume-safe ingestion pipeline                          |

```
CSV Emails
    ↓

    ↓

    ↓

    ↓
Artifact Hashing
    ↓
Neo4j Graph Storage
    ↓

```

---

## 📝 Notes & tips

* The extraction prompt is strict — adjust `prompts.py` for domain adaptation.
* Claims without valid evidence spans are discarded.
* `progress.json` ensures crash-safe ingestion.
* Claim identity is hash-based to prevent duplication.
* You can extend the QA layer with embeddings for semantic search.
